关于第二种，直接部署到服务器的方式，我这里有一份可以直接部署的deploy.yml文件：

```yaml
name: Deploy Vue App to My Windows Server

on:
  push:
    branches: [main]

permissions:
  contents: read

concurrency:
  group: 'deploy-to-server'
  cancel-in-progress: true

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build Vue App
        run: npm run build
        # 假设构建产物在 ./dist 目录

      # ====== SSH 配置部分 ======
      - name: Setup SSH agent
        uses: webfactory/ssh-agent@v0.5.3
        with:
          ssh-private-key: ${{ secrets.SSH_PRIVATE_KEY }}

      - name: Add server to known_hosts
        run: |
          mkdir -p ~/.ssh
          ssh-keyscan -H ${{ secrets.SERVER_IP }} >> ~/.ssh/known_hosts

      # ====== 清空远程目录并上传 ======
      - name: Clean remote directory and upload via SCP
        env:
          SERVER_IP: ${{ secrets.SERVER_IP }}
        run: |

          ssh ggcyuser@${SERVER_IP} "Remove-Item -Recurse -Force 'C:/test_ci_cd/*' -ErrorAction SilentlyContinue"

          scp -v -r -o StrictHostKeyChecking=no ./dist/* ggcyuser@${SERVER_IP}:"C:/test_ci_cd"

```

注意，这里我并没有使用别人写好的actions，因为我用了以后，发现一直连接不上我的服务器，我不清楚什么原因，是否和我的服务器是windows的有关；所以这里我使用了自己写脚本的方式，上面就是可以的；

注意配合nginx的配置，以及要注意放到服务器的哪个地方
